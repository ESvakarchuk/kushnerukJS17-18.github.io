Пояснения к шаблону Gulp

1. В папке "app" должны лежать файлы продакшена с которыми нужно работать
(в дальнейшем будут обработаны по команде и отправлены в папку "dist");

2. В файле "gulpfile.js" созданы, подключены и описаны все необходимые плагины и таски (команды);

3. В "package.json" указаны все плагины которое участвуют в проекте;

4. Bower загружает в продакшен необходимые для работы фреймворки (по команде), загружает он их в папку указанную в файле ".bowerrc";

5. В папке "css" есть файлы: "libs.min.css" и "main.css".
   "libs.min.css" - Это уже сжатый файл подключаемых библиотек, например "reset.css".
   Для того чтобы в него добавить библиотеку, ее нужно положить в папку "libs", и указать путь к ней в файле "sass/libs.sass"  -
   например: "@import "app/libs/magnific-popup/dist/magnific-popup" (Импортируем библиотеку Magnific Popup).

   В файл "main.css" автоматически импортируются стили описанные в файле "sass/main.sass".

   Т.е. в html просто указываем link на эти файлы, сначала "libs.min.css"(reser.css), затем "main.css"(главные стили страницы).
   Сами файлы в папке "css" трогать не нужно, они обрабатываются автоматически!

6. В папке "js", так же есть 2 файла "common.js" и "libs.min.js".
   В "libs.min.js" импортируются и сжимаются все дополнительные библиотеки из папки "libs" (JQuey например...), т.е. его подключаем к странице первым.
   В "common.js" мы пишем js для страницы (и как всегда подлючаем через script).

7. Все изображения в папке "img" будут автоматически сжаты и оптимизированы (при перемещении в папку "dist");
